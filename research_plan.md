## Concept of R&D & Business
기존의 고지능 대형 언어모델과는 다른 방향으로 언어모델을 개발해 보려고 해. 
reflexive language model(가칭) 곤충의 뇌처럼 단순하고 효율적이고 빠른 언어모델을 구현하려고 해. 
자연어 -> DSL -> function call로의 소프트웨어 파이프라인을 구성하여 자연어 -> function call 효율을 극대화 시키는게 목표야. 
바이브 코딩을 넘어서 범용 바이브 오퍼레이션 시스템을 구성하려는게 큰그림이야. 
또한 오퍼레이션의 구동 주체를 사람이 아닌 대형언어모델로 전환한다면 완전 자동화된 오퍼레이션 시스템 구동도 가능하다고 생각한다. 
여기서 언어모델의 역할은 자연어->DSL 변환용 컴포넌트야.

## Important idea
모델 소형화 및 constrained decoding 이용하여 극한의 가성비를 추구하려고 해. 
연구의 비전에 대해 객관적으로 평가하고 기존 상용언어모델대비 경쟁력이 있어보이는지 검토하고 
어느분야에서 우선적으로 적용해보면 좋을까 제안해봐.


## Important engineering part for business
여기서의 핵심은 DSL 스키마의 자동 생성이야. 고객(사)이 자기네쪽 실행가능 함수를 제시하면 
이 함수들과 대응되는 DSL 스키마를 자동으로 생성하고 이게 최적화된 모델을 즉석해서 학습 또는 튜닝하여 생성하는거야. 
이게 가능하겠냐?
또한 언어모델을 사용하여 자연어->함수로 바로 매핑하는것보다 자연어 -> DSL -> 함수매핑이 유리한지 검증할 필요가 있다


## 사업화가능여부 쟁점
1. 기존의 대형언어모델의 비용 비중이 오퍼레이션 총 비용대비 충분히 작은 비중이면 굳이 여기서 더 비용절감이 필요하지 않을거 같음.
2. 대형언어모델들의 비용이 충분히 싸고 function calling, mapping 능력이 충분히 좋다면 시장성이 없을 수 있음.


## POC
1. 기존 언어모델을 기반으로 자연어->DSL->function call 이 자연어->function call 보다 정확도에서 유리한지 조사
2. 자연어->DSL 특화 모델을 실제로 학습시키고 해당모델을 사용하여 특화전 모델보다 자연어->DSL 정확도 및 토큰 효율성 검토
3. function 명세서 제시핬을떄 DSL 스키마를 자동생성하고 해당 DSL 생성에 최적화된 모델 자동 생성하는 파이프라인 구현
